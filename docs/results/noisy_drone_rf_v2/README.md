# Noisy Drone RF v2 Result Card

This folder documents the canonical Noisy Drone RF Signal Classification v2 evaluation evidence generated by `notebooks/44_evaluation_noisy_drone_rf_v2.ipynb`. The matching training notebook is `notebooks/33_vgg_spectrogram_noisy_drone_rf_v2.ipynb`.

## Result Snapshot

| Field | Value |
|---|---|
| Dataset | Noisy Drone RF Signal Classification v2 |
| Data caveat | Public preprocessed/noise-augmented I/Q data; not over-the-air Navy field data |
| Model | VGG full-complex spectrogram classifier |
| Model path | `models/noisy_drone_rf_v2/noisy_drone_rf_v2_vgg_full_complex_spectrogram_best.keras` |
| Split method | Stratified 80/20 held-out test split with `NOISY_DRONE_MIN_SNR_DB=-6` and `NOISY_DRONE_DATA_FRACTION=0.25` |
| Seed | 1961 |
| Classes | 7 |
| Natural eval support | 649 held-out samples |
| Balanced eval support | 203 samples: 29 held-out windows per class |
| Natural accuracy | 0.9769 |
| Natural macro F1 | 0.9775 |
| Natural weighted F1 | 0.9767 |
| Balanced accuracy | 0.9803 |
| Balanced macro F1 | 0.9807 |
| Balanced weighted F1 | 0.9807 |

## Interpretation Notes

- "1 eval window" means one fixed-length spectrogram window per source `.pt` sample after trimming/windowing to the configured evaluation length. It does not mean only one total evaluation example.
- The visible weakness in the local run is small: Noise precision is 0.88, so a few drone/controller samples are predicted as noise, and DJI recall is 0.93.
- The committed SNR accuracy curve appears strong from low to high SNR.
