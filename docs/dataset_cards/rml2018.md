# Dataset Card: RML2018.01A

## Source

RadioML 2018.01A dataset, loaded from `GOLD_XYZ_OSC.0001_1024.hdf5` with local class-order files `classes.txt` and `classes-fixed.txt`.

## Class List

24 modulation classes from the local class files. The exact order must be read from `classes-fixed.txt` for reported repo metrics.

## SNR Range

The repo config evaluates a filtered range from `-6 dB` to `30 dB`. The full dataset includes broader SNR coverage; use the HDF5 `Z` field as the source of truth.

## Sample Count

The full public dataset is large-scale. The repo config uses:

- `max_samples_per_class=3000`
- `test_size=0.20`
- optional `max_per_class_per_snr=200` for evaluation slicing

The README records a legacy 72,000-sample evaluation snapshot and a current filtered checkpoint accuracy row.

## Preprocessing Assumptions

- I/Q comes from HDF5 dataset `X`.
- One-hot labels come from `Y`.
- SNR metadata comes from `Z`.
- The LSTM workflow appends SNR as a third feature channel across the time axis.

## Leakage Risks

- Class-order mismatch between `classes.txt`, `classes-fixed.txt`, and model outputs is a major risk.
- SNR filtering changes the evaluation population.
- Cap-per-class sampling should be reported when used.

## Recommended Evaluation Split

Use `configs/rml2018_eval.yaml` and report:

- class-order file used
- SNR filter
- max samples per class
- test split seed
- sample count
