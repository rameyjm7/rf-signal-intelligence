# Docker and HPC Runtime

This directory contains container tooling for local GPU development (Docker) and HPC execution (Apptainer).

## Files

- `Dockerfile`: CUDA + cuDNN runtime image with Python dependencies and project package installation.
- `Makefile`: Build, run, push, and Apptainer conversion helpers.

## Prerequisites

- Docker with NVIDIA runtime support (`--gpus all`)
- Access to NVIDIA drivers compatible with CUDA 12.x
- Optional for HPC: `apptainer` module/installation

## Quick Start (Local Docker)

Run from the `docker/` directory:

```bash
make build
make run
```

This starts Jupyter Lab on port `8888`:

`http://localhost:8888`

For an interactive shell:

```bash
make shell
```

## Push Image

```bash
make push
```

By default, the image name is controlled in `Makefile`:

- `DOCKERHUB_USER`
- `NAME`
- `IMAGE_TAG`

## Apptainer / HPC

Build `.sif` from the Docker Hub image:

```bash
make asif
```

Run Jupyter Lab in Apptainer:

```bash
make apptainer-lab
```

Interactive shell in Apptainer:

```bash
make shell-hpc
```

Run the project CLI non-interactively:

```bash
apptainer exec --nv ml-wireless-signal-classification-hpc.sif \
  python3 -m ml_wireless_classification --mode evaluate_only
```

## Cleanup

```bash
make clean
```

## Notes

- The image embeds the current git short hash via `org.opencontainers.image.revision`.
- Project code is copied from `docker/code.tar.gz`, generated by `make build`.
- Data/model directories are mounted from the repository root when using `make run` and `make shell`.
